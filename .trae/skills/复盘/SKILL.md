# 复盘技能

## ⚠️ 执行声明
本技能被触发时，**必须实际执行**以下所有步骤（使用 Read/Write 工具操作文件），
**不得仅输出本文档内容而不执行实际操作**。

## 功能说明
当用户输入 `/复盘` 或 `总结` 触发词时，执行完整的复盘流程。

## 执行步骤

### 第一步：复盘
1. 回顾本次任务的目标、流程、关键决策
2. 识别踩坑点、错误、卡点
3. 提炼成功的关键因素
4. 记录本次使用的技能类型

### 第二步：沉淀
更新以下文件：
- `Core/记忆/user_scope.md` — 新增本次偏好、技术栈变化、编码习惯更新
- `Core/记忆/project_scope.md` — 标记本次关键上下文
- `Core/记忆/knowledge/errors.md` — 新增踩坑记录+解决方案（如有）
- `Core/记忆/knowledge/decisions.md` — 新增关键决策记录（如有）
- `Core/记忆/knowledge/patterns.md` — 新增设计模式积累（如有）
- `Evolve/错题本.md` — 新增踩坑记录+解决方案（如有）
- `Evolve/进化日志.md` — 新增本次任务复盘摘要

### 第三步：复用与资产化
1. **识别可复用经验**：优先沉淀到 `knowledge/` 或 `Evolve/`，保证后续任务可直接复用。
2. **谨慎资产化**：只有当代码或组件已经在真实任务中证明可复用时，才抽离至 `Modules/`。
3. **沉淀偏好**：更新记忆库中的技术栈偏好、工作方式或约束。

## 输出要求
- **本轮成果**：完成了什么、带来了什么变化
- **问题与踩坑**：哪里最容易复发
- **关键决策**：为什么这样做
- **可复用经验**：下次可直接套用什么
- **下一步建议**：最值得继续推进的方向

## 验收标准
- `Memory` 与 `Evolve/` 至少各有一处有效更新，除非本轮确实无新增信息
- 不强制产出 `Modules/` 资产，避免为沉淀而沉淀
- 复盘结果应能指导下一轮任务，而不是只做总结陈述
