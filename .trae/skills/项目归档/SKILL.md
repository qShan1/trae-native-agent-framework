# 项目归档技能

## ⚠️ 执行声明
本技能被触发时，**必须实际执行**以下所有步骤（使用 Read/Write 工具操作文件），
**不得仅输出本文档内容而不执行实际操作**。

## 功能说明
当项目完成时，执行归档流程。

## 执行步骤
1. 确认项目名称、版本、归档原因和是否仍需保留恢复入口。
2. 检查目标归档路径是否已存在，避免覆盖已有成果。
3. 将项目移动或整理到 `Builds/项目名称/`；如有版本，优先使用 `项目名称-vX.Y.Z/`。
4. 更新 `Core/记忆/project_scope.md` 中的项目状态、完成内容和后续入口。
5. 如该项目沉淀了长期偏好或经验，补写 `user_scope.md`、`knowledge/decisions.md`。
6. 输出归档结果、目录位置、恢复方式和是否还有未归档资产。

## 输出要求
- **归档对象**：项目名、版本、归档原因
- **归档位置**：最终路径
- **记忆更新**：同步了哪些长期信息
- **恢复入口**：以后如果要继续，从哪里开始

## 验收标准
- 归档目录真实存在
- `project_scope.md` 已将状态切换为完成/归档
- 用户能直接根据回复找到归档位置和恢复入口
