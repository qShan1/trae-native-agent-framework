# 测试技能

## ⚠️ 执行声明
本技能被触发时，**必须实际执行**以下测试/修复流程，
**不得仅输出本文档内容而不执行测试操作**。

## 触发条件
用户输入 `/测试` 或 `/修复` 或检测到需要验证时。

## 职责
- 质量验证与代码审查
- Bug定位与根因分析
- 测试用例设计与执行
- 性能测试与优化建议
- 问题分析与修复

## 执行步骤
1. 获取原始问题表现：报错、复现路径、诊断信息或用户反馈。
2. 使用 `GetDiagnostics`、`CheckCommandStatus`、`Read`、`Grep` 等工具定位根因。
3. 判断问题类型：语法、逻辑、配置、依赖、环境或需求误解。
4. 在当前任务范围内执行修复，并记录尝试过的路径。
5. 重新执行相关验证：`GetDiagnostics`、构建、测试命令或页面检查。
6. 输出结论时说明已修复内容、验证范围和残留风险。

## 输出要求
- **问题现象**：用户实际遇到什么
- **根因分析**：为什么会发生
- **修复动作**：改了什么
- **验证结果**：如何确认问题消失
- **风险说明**：哪些点仍未覆盖

## 验收标准
- 至少完成一次“复现/诊断/验证”闭环
- 不能只报错不修，或只猜测不验证
- 如果未完全解决，必须明确卡点和下一步建议
